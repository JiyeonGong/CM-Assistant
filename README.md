# CM Assistant

행정팀의 단위기간 출석부를 로컬에서 분석하고 Slack 보고 멘트를 생성하며, 오늘의 업무 To-do를 관리하는 데스크톱 앱입니다.

## 기술 스택

- Electron
- React
- TypeScript
- exceljs
- electron-vite
- electron-builder

## 1차 MVP 범위

- `단위기간 출석부.xlsx` 파일 선택 또는 엑셀 표 붙여넣기
- `훈련중` 수강생 기준 오늘 출결 집계
- 출석, 지각, 외출, 조퇴, 휴공가, 결석, 출석률 표시
- 퇴실 QR 미촬영 자동 집계
- 오전 출결 보고 멘트 생성
- 오후 출결 보고 멘트 생성
- 최종 확정 출결 보고 멘트 생성
- 클립보드 복사
- Dashboard에서 오늘 업무 확인
- Todo 직접 입력, 상태 변경, 삭제
- Dashboard 빠른 업무 추가
- 오늘 루틴 업무 자동 생성
- Todo 로컬 JSON 저장
- 자주 쓰는 멘트 모음 메뉴
- 개인정보 서버 전송 없음

## 현재 분석 기준

- 기준 데이터: `단위기간 출석부.xlsx` 파일 또는 엑셀에서 복사한 단위기간 출석부 표
- 붙여넣기 방식은 엑셀 전체 선택 복사 시 생기는 탭 구분, 따옴표, 셀 내부 줄바꿈을 처리합니다.
- 집계 대상: `훈련생 상태`가 `훈련중`인 행
- 출석 인원: `출석`, `지각`, `외출`, `조퇴` 상태를 포함
- 지각: 입실 시간이 `09:10` 이상이면 지각으로 분류
- 조퇴: 퇴실 시간이 있고 `18:50` 이전이면 조퇴로 분류
- 외출: 출결상태가 `외출`인 경우 외출 시작/복귀 시간을 표시
- 결석: 출결상태가 `결석`이거나 `100분의50미만`/`50미만`을 포함하면 결석으로 분류
- 퇴실 QR 미촬영: 출석, 지각, 외출 상태인데 퇴실 시간이 비어 있으면 QR 미촬영으로 분류
- 오전 보고: 입실 시간이 비어 있는 사람은 `부재중`으로 표시
- 중도탈락 등 `훈련중`이 아닌 행은 전체 인원에서 제외
- 출석입력요청은 처리상태가 `-`가 아닌 행을 검토 대상으로 집계
- 분석 완료 시 오전 보고 멘트가 자동 생성됩니다.
- 출석부 분석 전에도 오전/오후/최종 보고 버튼으로 수강생 정보가 없는 기본 보고 양식을 생성하고 복사할 수 있습니다.

## Todo / Dashboard

- 상단 탭에서 `Dashboard`, `Attendance`, `Todo` 화면을 전환합니다.
- `Todo` 화면에서 할 일을 추가하고 상태를 변경할 수 있습니다.
- `Dashboard` 상단 입력창에서 생각난 업무를 바로 추가할 수 있습니다.
- 앱 실행 시 오늘 날짜의 기본 루틴 업무가 자동 생성됩니다.
- 앱을 계속 켜둔 상태에서도 날짜 변경을 감지해 새 날짜의 루틴 업무를 생성합니다.
- 루틴 업무는 날짜별로 새로 생성되며, Dashboard에는 오늘 날짜의 루틴만 표시됩니다.
- 직접 추가한 Todo는 완료하거나 삭제하기 전까지 다음 날에도 계속 표시됩니다.
- 자동 생성된 업무는 `루틴`, 직접 추가한 업무는 `추가`로 구분됩니다.
- Dashboard의 추가 업무는 `×` 버튼으로 바로 삭제할 수 있습니다.
- Dashboard의 루틴 업무는 체크박스로 바로 완료 처리할 수 있습니다.
- Dashboard의 `진행` 버튼을 누르면 해당 업무가 `진행 중` 영역에 표시됩니다.
- 상태값은 `진행 전`, `진행 중`, `완료`, `보류`입니다.
- `Dashboard`는 오늘 해야 할 일, 진행 중 업무, 오늘 완료한 업무, 전체 상태 요약을 표시합니다.
- Todo 데이터는 서버로 전송하지 않고 Electron `userData` 경로의 `cm-assistant-data.json`에 저장합니다.

## 자주 쓰는 멘트

- 상단 탭의 `Messages`에서 자주 쓰는 멘트를 선택할 수 있습니다.
- 멘트 목록은 `불시점검`, `인사` 카테고리로 구분됩니다.
- 현재는 불시점검 안내, 불시점검 후속 DM, 불시점검 결과 보고, 출결현황공유(주강사), 아침 인사, 저녁 인사 멘트를 제공합니다.
- 불시점검 안내 멘트는 현재 시각과 15분 뒤 복귀 기준 시각을 자동 반영합니다.
- 불시점검 안내 멘트는 디스코드 DM과 Zep 채팅 문구를 분리해 각각 복사할 수 있습니다.
- 불시점검 후속 DM은 현재 시각과 15분 전 최초 안내 시각을 자동 반영합니다.
- 불시점검 결과 보고는 오늘 날짜와 요일을 자동 반영하고, 기수/교시/특이사항 수는 직접 수정합니다.
- 출결현황공유(주강사), 아침/저녁 인사 멘트는 각자 문구를 수정해 저장할 수 있고, 다음 실행 때도 저장된 문구를 불러옵니다.
- 출결현황공유(주강사)는 디스코드 마크다운 원문을 그대로 복사합니다.
- 멘트 선택 후 내용을 수정하고 `복사하기`로 Slack에 붙여넣을 수 있습니다.
- 실제 운영 양식은 추후 확정되는 문구로 교체할 예정입니다.

## UX/UI 개편 방향

- 현재 기능 MVP를 기준점으로 두고, 다음 단계에서 UX/UI를 정리합니다.
- 전체 톤은 Toss처럼 간결하고 명확한 화면 흐름을 목표로 합니다.
- 단순히 색상과 카드 스타일만 바꾸지 않고, 화면별 핵심 행동을 줄이는 방향으로 개편합니다.
- `Dashboard`는 오늘 처리해야 할 루틴과 추가 업무를 가장 먼저 보여주는 홈 화면으로 강화합니다.
- `Todo`는 Dashboard와 역할이 겹치지 않도록 전체 업무 관리/정리 화면으로 단순화합니다.
- `Attendance`는 `출석부 입력 → 분석 결과 확인 → 보고 멘트 복사` 순서가 자연스럽게 보이도록 단계형 UX를 검토합니다.
- `Messages`는 템플릿 보관함보다 바로 복사해서 쓰는 도구에 가깝게 정리합니다.
- `Attendance`는 화면 전환 없이 `Step 1 출석부 넣기`, `Step 2 결과 확인`, `Step 3 보고 복사` 흐름으로 정리합니다.
- `Messages`는 `Step 1 멘트 선택`, `Step 2 확인하고 복사하기` 흐름으로 정리합니다.
- 기능별 참고 방향은 `Todoist + Process Street`(Dashboard/Todo), `Airtable + PowerSchool`(Attendance), `Text Blaze + Notion Templates`(Messages)입니다.
